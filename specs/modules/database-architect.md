# Module: Database Architect Agent

This is the third agent in the Chain of Responsibility. Its job is to design 
the database schema and generate an Entity-Relationship Diagram (ERD) based 
on the business requirements and user stories.

## 0. Project Context
- Base package: com.autonomouspm
- Models package: com.autonomouspm.models
- Agent package: com.autonomouspm.agents.database
- Infrastructure package: com.autonomouspm.infrastructure
- Language: Java 21, Spring Boot 3.5.0, LangChain4j 1.0.0-beta3
- DTOs: Java Records only — no regular classes for data
- Logging: SLF4J only — zero System.out.println anywhere
- All LLM calls: through CachingAiService only — never call AiService directly
- Token budget for this agent: 1000 max output tokens

## 1. Input Specifications
- Data Source: BusinessContext and RequirementContext, provided via Mediator.
- Before calling the LLM, InputStripper must extract ONLY these fields 
  from BusinessContext:
    - userStories
    - coreFeatures
    - constraints
  Do NOT forward executiveSummary, marketInsights, or any other field.
  This is mandatory for token cost management.

## 2. Core Logic
- Connect to CachingAiService (never AiService directly).
- Instruct the LLM to act as a Senior Database Architect.
- Derive entities from:
  1. Explicit business objects mentioned in requirements.
  2. Entities required to support identified features.
  3. Existing constraints from RequirementContext.
- Do not create entities that cannot be justified by a requirement or user story.
- Publish state updates ("Designing Schema...", "Generating ERD...") 
  via EventLogger (Observer pattern).

## 2.1 Schema Validation Rules
The Database Architect must:
- Generate only entities supported by requirements.
- Avoid speculative entities.
- Ensure every relationship has a business justification.
- Ensure every table has a primary key.
- Resolve many-to-many relationships using junction tables when appropriate.
- Prefer normalized schema design (up to 3NF unless requirements dictate otherwise).
- Flag ambiguities instead of inventing assumptions.

The Mermaid ERD is a secondary visualization.
Tables and relationships are the authoritative schema definition.

## 3. Output Specifications
The LLM response MUST be parsed into a DatabaseContext Java Record.

DatabaseContext must contain:
- List<Table> tables
- List<Relationship> relationships
- String mermaidErdChart (generated by MermaidErdGenerator, NOT by the LLM)

Records to create:

public record DatabaseContext(
    List<Table> tables,
    List<Relationship> relationships,
    String mermaidErdChart
) {}

public record Table(
    String name,
    String description,
    List<Column> columns
) {}

public record Column(
    String name,
    String dataType,
    boolean isPrimaryKey,
    boolean isForeignKey,
    boolean isNullable
) {}

public record Relationship(
    String fromTable,
    String toTable,
    String cardinality,
    String justification
) {}

On LLM failure: return a Null Object (EmptyDatabaseContext) with empty 
lists and mermaidErdChart = "" — never return null.

## 4. MermaidErdGenerator
Create MermaidErdGenerator.java in com.autonomouspm.infrastructure.
This class takes List<Table> and List<Relationship> and produces 
a Mermaid ERD string like:

erDiagram
  CUSTOMER ||--o{ ORDER : places
  ORDER ||--|{ ORDER_ITEM : contains

Rules:
- Table names uppercased in Mermaid output
- Cardinality mapping:
    ONE_TO_ONE   → ||--||
    ONE_TO_MANY  → ||--o{
    MANY_TO_MANY → }o--o{
- This is pure Java logic — no LLM involved

## 5. DatabaseArchitectValidator
Create DatabaseArchitectValidator.java in com.autonomouspm.agents.database.
Validate:
- At least one table exists
- Every table has at least one column
- Every table has exactly one primary key column
- Every relationship references tables that exist in the table list
- No duplicate table names
On failure: log warnings via SLF4J and return the best partial result,
never throw and never return null.

## 6. Execution Directives
Build files in this exact order, one at a time:
1. Column.java (com.autonomouspm.models)
2. Table.java (com.autonomouspm.models)
3. Relationship.java (com.autonomouspm.models)
4. DatabaseContext.java (com.autonomouspm.models)
5. MermaidErdGenerator.java (com.autonomouspm.infrastructure)
6. DatabaseArchitectValidator.java (com.autonomouspm.agents.database)
7. DatabaseArchitectAgent.java (com.autonomouspm.agents.database)

### Restrictions:
- Do not invent tables without justification.
- Do not add any Maven dependency not already in pom.xml.
- Do not create any file not listed above.
- Every table must be traceable to at least one requirement or user story.
- If information is insufficient, mark entity description as 
  "Needs Clarification" instead of inventing a solution.
- SLF4J logging only — no System.out.println anywhere.
- All data types as Java Records — no regular classes.
